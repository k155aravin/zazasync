Auth setup Supabase Auth

Connect zazasync-auth.html to Supabase

The sign-in and sign-up forms are fully designed. Wire them to Supabase Auth:

Sign up flow

Library@supabase/supabase-js
Methodsupabase.auth.signUp({ email, password })
After signupRedirect to /onboarding — do NOT skip this step, it captures monetization data
Age gateThe 21+ checkbox must be checked or form does not submit — already in HTML, just enforce server-side too

Sign in flow

Methodsupabase.auth.signInWithPassword({ email, password })
Google OAuthsupabase.auth.signInWithOAuth({ provider: 'google' }) — enable Google provider in Supabase dashboard
After signinRedirect to /watchlist if onboarding done, else redirect to /onboarding

Alerts system Email first, SMS later

Email alerts — how it works

TriggerWhen the SQDC crawler detects a product flipping from out-of-stock → in-stock
LogicQuery watchlist table for all users watching that product_id with alert_enabled = true
Email serviceUse Resend.com — free tier, simple API, works perfectly with Supabase Edge Functions
Log itWrite a row to alert_log every time an alert fires

SMS alerts — phase 2

ServiceTwilio — only add after email alerts are stable and user demand confirmed
UIAlready grayed out in the watchlist settings panel — just enable the toggle when ready

Event tracking Do not skip — this is the money

Install PostHog (free, self-hostable) alongside the existing Supabase setup. Track these events at minimum:

searchEvery search query + filters used + result count
product_viewEvery product detail page opened
watchlist_addProduct added to watchlist — highest purchase intent signal
alert_setUser sets a back-in-stock alert
filter_usedWhich filter chips get clicked (In stock / Pre-rolls / Under $25 etc)
store_viewWhich store pages get viewed

All events must include: user_id, age_range, region, language, timestamp. This is what gets exported monthly for brand intelligence reports.

Monthly data export Owner uses this to make money

The owner needs to export data monthly to analyze and sell to cannabis brands. Set up a simple Supabase SQL view or scheduled export:

Export formatCSV downloadable from Supabase Table Editor — no custom dashboard needed at first
Key queriesTop searched products by region / age range · Most watchlisted products · Alert volume by product · Visit frequency breakdown
AccessOwner logs into Supabase dashboard → Table Editor → Export to CSV. Done.

Legal requirements Quebec Law 25 — mandatory

Privacy policyMust state that anonymized, aggregated behavioral data may be shared with third parties for market research. Already linked in footer of all pages.
Consent at signupThe "I agree to Terms" checkbox at signup counts as consent. Keep a timestamp of when user agreed.
Law 25Quebec's privacy law — stricter than federal PIPEDA. Users can request deletion of their data. Add a "Delete my account" option in settings.
Age verificationSelf-declared 21+ checkbox is sufficient for MVP. Do not store government ID data.
Data soldOnly sell AGGREGATED and ANONYMIZED data. Never sell individual user profiles. This protects both the business and users.
